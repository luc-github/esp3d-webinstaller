🚀 New Features

grblHAL Code Base Integration (Work in Progress)

Integrated the grblHAL code base as the foundation for future grblHAL support alongside FluidNC. Initial connection flow and telnet/TCP-IP communication groundwork are in place.

[!WARNING]

grblHAL support is not yet complete. Connection token handling and dedicated UI screens are still under active development.

🐛 Bug Fixes

  • WiFi Connection Stability: Fixed TCP_NODELAY setting for WiFi connections and improved memory fragmentation handling to prevent connection drops.
  • SD Card Update Service: Fixed NO_SD flag handling in update service to properly manage firmware updates when SD card is unavailable.
  • Telnet Connection Flow: Fixed telnet connection establishment and updated communication flow for proper grblHAL compatibility.
  • Touch Input Edge Cases: Refactored pin keying process to properly handle edge cases in touch input detection.

💪 Improvements

  • Connection Flow Architecture: Rewrote connection flow for improved clarity and flexibility, including a full WiFi module refactoring. Enhanced screen transition system using consistent messagebox model to reduce boilerplate code.
  • mDNS Service Discovery: Added proper mDNS entries and fixed related glitches for better network device discovery.
  • Factory Button Configuration: Updated factory initialization with enhanced button handling and configuration.
  • Flash Tools: New interactive flash manager for easier firmware flashing.

📝 Other Changes

  • New Logo for 240×320: Updated visual asset for the native Pibot pendant display resolution.

[!NOTE]

grblHAL support is not part of this release. The code base integration is included as groundwork only. Full grblHAL support — including connection token handling and dedicated UI screens — is planned for the next release.
